# Claude 기본 워크샵

Anthropic의 **Claude** 를 처음 써보는 분들을 위한 가이드입니다.
가입 → 앱 설치 → 채팅 → 프로젝트 → 루틴 → Cowork / Scheduled / Dispatch 까지 순서대로 따라와 주세요.

> 터미널에서 코드를 직접 다루는 **Claude Code** 는 [[claude-code-workshop]] 에서 별도로 다룹니다.
> 이 문서는 "일반 사용자용 Claude (claude.ai 웹 / 데스크톱 앱)" 가이드입니다.

---

## 1. Claude 가입하고 앱 다운로드하기

### 1-1. 계정 만들기

1. 웹 브라우저에서 [https://claude.ai](https://claude.ai) 에 접속합니다.
2. 오른쪽 위 **`Sign up`** 버튼을 누릅니다.
3. 가입 방식 선택
   - **Google 계정으로 가입** (가장 쉬움, 추천)
   - 이메일 + 비밀번호로 가입 → 메일함에서 인증 링크 클릭
4. 이름, 사용 목적 등 간단한 정보를 입력하면 완료.

> 처음 가입하면 **Free 플랜**으로 시작됩니다. 무료로도 충분히 써볼 수 있고,
> 더 오래/더 많이 쓰고 싶다면 **Pro ($20/월)** 로 업그레이드하면 됩니다.
> (설정 → `Settings` → `Billing` 에서 업그레이드)

### 1-2. 데스크톱 앱 다운로드

브라우저에서도 쓸 수 있지만, **데스크톱 앱**을 깔면 훨씬 편합니다.
(단축키 호출, 화면 캡처 공유 등이 쉬워집니다.)

1. [https://claude.ai/download](https://claude.ai/download) 에 접속합니다.
2. 운영체제에 맞는 버튼을 클릭합니다.
   - **macOS** → `.dmg` 파일 다운로드
   - **Windows** → `.exe` 설치 파일 다운로드
3. 설치 후 실행 → 1-1 에서 만든 계정으로 **로그인**합니다.
4. 로그인이 끝나면 브라우저와 똑같은 채팅 화면이 앱에서 뜹니다.

> 모바일도 있습니다. App Store / Google Play 에서 **"Claude"** 검색 → 설치.

## 2. Claude 제품군 한눈에 보기

"Claude" 는 하나의 제품이 아니라 **여러 도구의 묶음**입니다.
이번 워크샵에서 전부 쓰진 않지만, 이름 정도는 알아두면 좋습니다.

| 이름 | 어디서 쓰나 | 무엇을 하나 |
|---|---|---|
| **Claude (Chat)** | 웹 `claude.ai`, 데스크톱/모바일 앱 | 가장 기본이 되는 **채팅 인터페이스**. 질문하고 답 받기, 문서 작성, 요약, 번역 등 |
| **Claude Desktop** | macOS / Windows 앱 | Chat 을 네이티브 앱으로 쓰는 것. 파일 드래그&드롭, 시스템 통합이 더 편함 |
| **Claude Cowork** | 웹 (claude.ai) | 여러 사람이 **같이 쓰는 협업 공간**. 공유 프로젝트, 공유 대화 |
| **Claude Code** | 터미널 / VSCode / JetBrains | 개발자용. 내 코드베이스를 직접 읽고 수정하는 **AI 코딩 에이전트** |
| **Claude API** | Anthropic Console | 개발자가 자기 앱에 Claude 를 **붙일 때** 쓰는 키/엔드포인트 |

> 이 워크샵에서는 **Chat** + **Desktop 앱** 사용법을 중심으로 보고,
> 마지막에 **Cowork / Scheduled / Dispatch** 같은 고급 기능을 훑어봅니다.

## 3. Chat / Desktop 앱 기본 사용법

웹이든 데스크톱이든 화면 구성은 거의 같습니다.

### 3-1. 화면 구성

```
 ┌────────────────────────────────────────────┐
 │  ≡   [New chat]                            │   ← 왼쪽: 대화/프로젝트 목록
 │  - 오늘                                     │
 │    · 여행 일정 짜기                         │
 │    · 이력서 리뷰                            │
 │  - 지난 7일                                 │
 │    · ...                                    │
 │                                            │
 │  ────────────────────────────              │
 │  Projects                                  │
 │    📁 회사 업무                              │
 │    📁 개인 공부                              │
 │                                            │
 │  [Settings]                                │
 └────────────────────────────────────────────┘
                 ↑
       중앙은 실제 "대화창"
       아래는 입력창 + 첨부 버튼
```

- **좌측 사이드바**: 지난 대화 목록, 프로젝트, 설정
- **중앙**: 현재 대화(Chat) 화면
- **하단 입력창**: 여기 메시지를 입력. `📎` 버튼으로 파일/이미지 첨부 가능

### 3-2. 첫 대화 해보기

하단 입력창에 한글로 그냥 질문하면 됩니다.

```
오늘 점심 뭐 먹을지 5개 추천해줘. 매운 건 빼고.
```

```
이 PDF 요약해줘.  ← 파일을 첨부(📎)하고 물어봐도 됨
```

```
영어 이메일 초안 좀 써줘. 받는 사람은 거래처 담당자고,
다음 주 미팅을 금요일로 옮기고 싶다는 내용이야.
```

### 3-3. 파일/이미지 첨부

- 입력창의 **📎 (클립)** 아이콘을 누르거나
- 파일을 **드래그해서 대화창에 떨어뜨리면** 됩니다.
- 지원: PDF, Word, Excel, 이미지(PNG/JPG), 텍스트 파일 등

### 3-4. 모델 선택

입력창 근처에 **모델 선택 드롭다운**이 있습니다.

| 모델 | 특징 | 언제 쓰나 |
|---|---|---|
| **Opus 4.6** | 가장 똑똑함, 느릴 수 있음 | 어려운 추론, 긴 글, 복잡한 분석 |
| **Sonnet 4.6** | 균형잡힘 (기본값) | 일상적인 질문 대부분 |
| **Haiku 4.5** | 제일 빠르고 가벼움 | 간단한 요약, 빠른 답 |

> 잘 모르면 **Sonnet** 그대로 쓰세요.

### 3-5. 대화 관리

- **New chat** (왼쪽 위 `+`) : 새 대화 시작. 주제가 바뀌면 새로 여는 게 좋음.
- **이름 바꾸기** : 대화 목록에서 우클릭 → `Rename`
- **삭제** : 우클릭 → `Delete`
- **공유** : 대화 우측 상단 `Share` 버튼 → 링크로 다른 사람에게 공유

## 4. Project — 맥락을 저장해두는 "작업 폴더"

### 4-1. Project 가 뭔가요?

Project 는 **"이 대화들은 같은 맥락이야"** 라고 Claude 에게 알려주는 **폴더**입니다.

- 관련된 **파일들**(PDF, 문서, 코드 등)을 한 번 올려두면
- 그 프로젝트 안의 **모든 대화**에서 Claude 가 그 파일들을 기억합니다.
- 매번 같은 자료를 다시 첨부할 필요가 없어집니다.

> 예시:
> - 📁 `논문 리서치` — 참고 논문 PDF 10개를 올려두고, 여러 대화에서 물어봄
> - 📁 `우리 회사 제품` — 제품 스펙 / FAQ 문서를 올려두고, 고객 응대 초안 생성
> - 📁 `이직 준비` — 내 이력서 / 커리어 정리본을 올려두고, JD 분석시키기

### 4-2. Project 만들기

1. 왼쪽 사이드바 `Projects` → **`+ New project`**
2. 이름을 짓습니다. (예: `영어 공부`)
3. **Project instructions** 에 이 프로젝트의 성격을 써둡니다.
   ```
   나는 영어 회화 초급이야.
   답은 한국어로 해주고, 영어 예문은 B1 수준으로.
   교정해줄 때는 원인도 같이 설명해줘.
   ```
4. **Project knowledge** 에 관련 파일을 업로드합니다. (PDF, 단어장, 스크립트 등)
5. 완료!

이제 이 Project 안에서 새 대화를 열면, 위에 쓴 지침과 파일을 Claude 가 **항상 참고**합니다.

### 4-3. Project 팁

- Project 설명은 **짧고 구체적으로**. "무엇을, 어떤 말투로, 어떤 형식으로"
- 자료가 바뀌면 Project knowledge 의 파일을 **교체/추가**하면 됩니다.
- 개인 Project 와 Cowork(공유) Project 는 구분됩니다 → 6-1 에서 설명.

## 5. 루틴 — Claude 와 매일 잘 쓰는 법

초보자가 가장 많이 하는 실수: **"한 번에 다 시키려 함"**.
Claude 는 **대화형 도구**이기 때문에, 아래 루틴이 훨씬 잘 됩니다.

### 5-1. 하루 단위 루틴

```
① 아침 — 오늘 할 일 정리
   "오늘 할 일 5개인데 우선순위 좀 잡아줘: ..."

② 업무 중 — 막힐 때마다 물어보기
   - 초안 써달라고 하기
   - 쓴 글 고쳐달라고 하기 (원문 붙여넣기)
   - 자료 요약시키기

③ 저녁 — 회고
   "오늘 이런 일이 있었는데, 내일 뭘 개선하면 좋을까?"
```

### 5-2. 좋은 질문(프롬프트) 패턴

1. **역할 + 상황 + 원하는 형식**을 한 문장에 담기.
   ```
   너는 HR 담당자야. 내가 퇴사 의사를 정중하게 전하고 싶어.
   2주 후 퇴사 기준, 격식 있는 이메일로 한국어 초안 3개만 써줘.
   ```
2. **예시를 같이 주기**.
   ```
   이런 느낌으로 3개 더 만들어줘:
   "AI 는 도구다. 쓰는 사람이 칼을 휘두르는 것이다."
   ```
3. **단계별로 지시**.
   - 먼저 요약 → 이어서 "이 중 2번을 더 풀어줘"
4. **안 맞으면 고쳐달라고 말하기**.
   ```
   톤이 너무 딱딱해. 좀 더 친근하게, 이모지는 빼고.
   ```

### 5-3. "이럴 땐 새 대화 / 이럴 땐 이어서"

| 상황 | 새 대화? |
|---|---|
| 주제가 완전히 바뀜 | ✅ New chat |
| 같은 문서를 계속 다룸 | ❌ 이어서 |
| Claude 가 헷갈려 하기 시작함 | ✅ New chat (또는 같은 Project 안에서) |
| 너무 길어져서 느림 | ✅ New chat |

## 6. 고급 기능: Cowork / Scheduled / Dispatch

기본에 익숙해지면 써볼 기능들입니다.

### 6-1. Cowork — 팀/친구랑 **같이** 쓰기

**Cowork** 는 **공유 프로젝트 / 공유 대화** 공간입니다.

- 혼자 쓰는 Project 와 달리, **여러 명이 같은 Project 에 접근**할 수 있습니다.
- 팀원 A 가 자료를 올리면 팀원 B 도 그 자료를 쓰는 Claude 와 대화 가능.
- 대화 자체를 공유할 수도 있어, 누가 어떤 답을 받았는지 **같이 리뷰**할 수 있습니다.

사용법:

1. Project 를 만든 뒤 우측 상단 `Share` / `Invite`
2. 이메일로 팀원 초대 (권한: View / Edit 선택)
3. 초대받은 사람은 자기 계정에서 그 Project 가 보이게 됩니다.

> 업무용이라면 **Team 플랜** 이상을 권장 (관리자 기능, SSO 제공).

### 6-2. Scheduled — **예약 실행**

**Scheduled** 는 "Claude 야, 이 일을 **정기적으로** 대신 해줘" 기능입니다.

- 예: 매주 월요일 아침 9시에 주간 뉴스 요약 생성
- 예: 매일 저녁 6시에 내 할 일 회고 초안 작성
- 예: 매월 1일에 예산 리포트 템플릿 만들기

사용법(개념):

1. 원하는 작업을 채팅으로 먼저 한 번 해봅니다. (결과가 만족스러운지 확인)
2. 그 대화의 `⋯` 메뉴에서 **`Schedule`** (또는 입력창의 시계 아이콘)
3. 반복 주기(매일/매주/매월) + 시간 설정
4. 정해진 시간에 Claude 가 **자동으로 실행**하고, 결과를 해당 대화에 남겨둡니다.

> 결과를 이메일이나 Slack 으로도 받게 연결해둘 수 있습니다 (플랜/통합 설정에 따라 다름).

### 6-3. Dispatch — **외부로 보내기 / 자동화**

**Dispatch** 는 Claude 의 결과물을 **다른 서비스로 흘려보내는** 기능입니다.

- 예: Claude 가 만든 회의록 초안을 → **Notion 페이지**로 자동 업로드
- 예: Claude 가 정리한 이슈 목록을 → **Linear/Jira 티켓**으로 전송
- 예: Claude 가 쓴 답장을 → **Gmail 초안**으로 저장

사용법(개념):

1. `Settings` → `Connections` 에서 연동할 서비스(Notion, Slack, Gmail 등)에 로그인.
2. 대화에서 결과가 나오면, 우측 상단 `Dispatch` (또는 `Send to...`) 선택.
3. 목적지(예: 특정 Notion DB, 특정 Slack 채널) 지정 후 전송.

> Scheduled + Dispatch 를 합치면 **"매주 월요일 아침 주간 리포트를 만들어 Slack 에 올리기"** 같은 자동화가 됩니다.

## 7. Connector — Claude 를 **내 계정의 도구들과 연결**하기

**Connector** 는 Claude 를 **내가 쓰는 외부 서비스**(Gmail, Google Calendar, Notion, Slack, GitHub 등)와 연결해 주는 기능입니다.

- 연결해두면 Claude 가 그 서비스를 **직접 읽고/쓸** 수 있습니다.
  - "내 Gmail 에서 오늘 온 메일 요약해줘"
  - "다음 주 캘린더 비는 시간 찾아줘"
  - "이 내용 Notion `회의록` DB 에 새 페이지로 올려줘"
- Dispatch(결과 내보내기) 와 비슷해 보이지만, Connector 는 **읽기 + 쓰기 모두** 가능한 양방향 연결입니다.

### 7-1. Connector 연결하기 (예: Gmail)

1. 좌측 하단 `Settings` → **`Connectors`** (또는 `Connections`) 탭으로 이동.
2. 목록에서 **`Gmail`** 을 찾아 `Connect` 버튼 클릭.
3. 브라우저에 구글 로그인 창이 뜹니다 → 연결할 **Gmail 계정 선택**.
4. 권한 동의 화면에서 Claude 가 요청하는 권한(메일 읽기/초안 쓰기 등)을 확인 후 **허용**.
5. 완료되면 Connectors 목록에 **초록색 `Connected`** 표시가 뜹니다.

> 연결을 끊고 싶으면 같은 화면에서 `Disconnect`. 권한은 Google 계정 설정에서도 해지 가능.

### 7-2. 사용 예: 최신 메일 5개 요약 + 답장 초안

연결이 끝났으면, 채팅창에 그냥 이렇게 입력하면 됩니다.

```
내 Gmail 에서 **최신 메일 5개**를 가져와서
각각 보낸 사람 / 제목 / 핵심 내용 3줄 요약으로 정리해줘.

그리고 그 중에서 **답장이 필요한 메일**이 있으면,
한국어로 정중한 답장 **초안**도 같이 써줘.
(보내지는 말고 초안까지만)
```

Claude 는 이런 식으로 동작합니다.

1. Gmail Connector 로 최근 받은편지함 5개를 읽어옴
2. 각각을 정리해서 표/리스트로 보여줌
3. 답장이 필요해 보이는 메일에 대해 초안을 작성
4. (원하면) "이 초안을 Gmail **Drafts** 로 저장해줘" 라고 이어서 지시 → 실제 지메일 "임시보관함"에 저장됨

> 실제로 **전송**까지 하려면 한 번 더 명시적으로 시켜야 합니다. ("그대로 보내줘")
> 실수 방지를 위해 Claude 는 기본적으로 **초안까지만** 만듭니다.

### 7-3. 자주 쓰는 Connector 예시

| Connector | 할 수 있는 일 |
|---|---|
| **Gmail** | 메일 검색/요약, 답장 초안, 라벨별 정리 |
| **Google Calendar** | 일정 조회, 빈 시간 찾기, 회의 잡기 |
| **Google Drive** | 문서/스프레드시트 읽고 요약 |
| **Notion** | 페이지 검색, 새 페이지 생성, DB 항목 추가 |
| **Slack** | 특정 채널 메시지 읽기, 메시지 보내기 |
| **GitHub** | 이슈/PR 읽기, 코드 리뷰, 이슈 생성 |

> Connector 는 **플랜/조직 정책**에 따라 제공 범위가 다릅니다.
> 회사 계정이면 관리자가 허용한 것만 보일 수 있어요.

## 8. 설정에서 한 번쯤 봐둘 것들

좌측 하단 `Settings` 에서:

- **Profile** : 이름, 이메일, 비밀번호
- **Appearance** : 라이트/다크 모드
- **Account** : 플랜(Free/Pro) 확인, 결제
- **Connections** : Notion / Slack / Gmail 등 외부 연동 (Dispatch 용)
- **Data controls** : 내 대화를 모델 학습에 쓸지 말지 선택 (기본 Off 권장)
- **Keyboard shortcuts** : 단축키 목록

## 9. 자주 묻는 질문

- **Q. 한국어 잘 되나요?** → 네, 아주 잘 됩니다. 그냥 한국어로 쓰세요.
- **Q. 내가 올린 파일이 학습에 쓰이나요?**
  → Settings → `Data controls` 에서 끌 수 있습니다. Pro/Team 은 기본적으로 **학습에 안 씁니다**.
- **Q. 답이 이상해요.**
  → (1) 새 대화 열기 (2) 모델을 Opus 로 바꾸기 (3) 질문을 더 구체적으로.
- **Q. 데스크톱 앱과 웹, 뭐가 나아요?**
  → 기능은 같습니다. 단축키/파일 드래그 편하면 **데스크톱**, 회사 노트북 설치 제한 있으면 **웹**.
- **Q. Pro 꼭 필요한가요?**
  → Free 로도 충분히 맛보기 가능. 하루에 오래 쓸 거면 Pro 가 편합니다.
